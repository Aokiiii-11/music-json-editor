# 功能验证报告（初版）

## 符号保留测试
- 覆盖内容：包含 `|`、`丨`、`｜` 以及中英标点的字符串字段。
- 结论：在新流程中不再进行任何基于分隔符的截断，所有特殊符号均保持原样。
- 证据：`tests/jsonPath.test.ts` 的断言对 `C丨D` 原样保留通过。

## 逻辑安全性测试
- 遍历：采用迭代遍历（栈）与最大节点上限（1,000,000）防止栈溢出。
- 类型守卫：严格区分 `string/array/object`，仅对字符串进行处理。
- 结构匹配：`diagnoseMatch` 报告缺失、多余与类型不一致，导出前提示但不阻塞原文导出。

## 性能测试（初步）
- 规模：在单元测试环境进行基础路径收集与匹配，2 个文件、4 项断言，运行耗时约 315ms（含构建）。
- 说明：实际大规模 JSON 的性能将在后续补充基准脚本中评估，当前迭代遍历复杂度 O(N)。

## 兼容性
- 旧格式迁移：加载含 `英文 | 中文` 的 JSON 时，自动分离为原文与译文映射，编辑器显示参考译文，导出仅为原文。
- UI：保留原有可视化编辑器交互，新增译文输入框不影响原文编辑流程。

## 风险与缓解
- 译文缺失：导出译文时提示缺失计数；原文导出不受影响。
- 结构差异：类型不一致在诊断中列出，建议在用户修正后导出译文。
